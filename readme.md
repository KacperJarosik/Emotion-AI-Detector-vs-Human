# Analysis of the Impact of Wearing a Protective Mask on Facial Emotion Recognition Using Machine Learning

## About the Project
This project is the research component of an engineering thesis. Its main objective is to analyze the impact of covering the lower part of the face (wearing a protective mask) on the effectiveness of recognizing seven basic emotions using machine learning methods, and to compare these results with human perceptual capabilities. 

The research utilizes the RAF-DB (Real-world Affective Faces Database) dataset. The analysis is based on three approaches:
* Original images (without a mask).
* Images with a generated synthetic surgical mask.
* Vector data (facial landmarks) extracted using the MediaPipe Face Landmarker model.

During the experiments, the effectiveness of various classifiers was tested, including classical algorithms on numerical data (e.g., XGBoost, Extra Trees, SVM, MLP) and Deep Convolutional Neural Networks (CNN) on image data. The results of the artificial intelligence models were compared with the results of a survey conducted on a group of 103 participants. 

## Dataset

The project utilizes a preprocessed subset of the **Real-world Affective Faces Database (RAF-DB)**. 

- **Original Data**: The initial dataset consists of high-quality facial images from the RAF-DB, normalized to 100x100 pixels.
- **Synthetic Masked Data**: To study the effect of occlusion, a synthetic dataset was generated by programmatically overlaying a surgical mask graphic onto each face image. This ensures a controlled comparison between masked and unmasked scenarios.
- **Vector Data**: Using Google's MediaPipe Face Landmarker, each unmasked image was converted into a numerical representation consisting of 468 3D facial landmarks and 1,322 connecting vectors. For the masked vector data, landmarks and vectors corresponding to the occluded facial area were zeroed out to simulate data loss while maintaining structural consistency.

## Key Results

### AI Model Evaluation

The AI models were evaluated on the test set across four scenarios. The CNN trained on unmasked images achieved the highest performance on unmasked faces, while the XGBoost model trained on masked data demonstrated superior robustness when tested on masked faces.

| Model             | Test Dataset | Micro Accuracy | Macro F1-Score |
|-------------------|--------------|----------------|----------------|
| **CNN (No Mask)**   | No Mask      | **77.0%**      | 67.1%          |
| CNN (No Mask)     | With Mask    | 51.8%          | 39.0%          |
| **XGBoost (Mask)**  | With Mask    | **60.8%**      | 48.8%          |
| XGBoost (Mask)    | No Mask      | 60.8%          | 48.8%          |

### Human Performance Evaluation

Analysis of the survey data revealed a significant drop in human accuracy when faces were masked.

| Condition    | Micro Accuracy | Macro F1-Score |
|--------------|----------------|----------------|
| Without Mask | 70.0%          | 68.6%          |
| With Mask    | 48.5%          | 48.1%          |

### Visualizations

The notebooks contain detailed visualizations, including:
- **Confusion Matrices**: To compare prediction patterns between AI and humans.
- **Feature Importance**: Identifying which facial vectors were most influential for the XGBoost model's predictions.
- **SHAP Analysis**: Explaining the CNN's predictions by highlighting the most important pixel regions.

## File Structure

The project is divided into interactive Jupyter Notebooks, each responsible for a specific stage of the data processing and analysis pipeline:

```text
📂 ROOT
 ├── 📄 0 data preparation.ipynb               # Initial processing of the RAF-DB dataset and generation of synthetic masks
 ├── 📄 1 data exploration.ipynb               # EDA, class balance analysis, and visualization of facial vectors
 ├── 📄 2 training from images data (CNN).ipynb # Definition and training of the Convolutional Neural Network (CNN)
 ├── 📄 3 training from vectors data (XGBoost).ipynb # Training the XGBoost classifier on landmark data
 ├── 📄 4 models evaluation.ipynb              # Evaluation of models (F1, ROC AUC, Confusion matrices, SHAP)
 ├── 📄 5 - form evaluation.ipynb              # Analysis of human emotion recognition results (survey)
 │
 ├── 📂 DATA                                   # Main data folder
 │    ├── 📂 FORM                              # Survey result sheet
 │    ├── 📂 IMAGES                            # Facial image datasets
 │    │    └── 📂 WITHOUT MASK                 # (Folder to be downloaded externally)
 │    │    └── 📂 WITH MASK                    # (Generate by "0 data preparation.ipynb")
 │    ├── 📂 OTHER                             # Auxiliary files
 │    ├── 📂 VECTORS                           # Numerical data (Generate by "0 data preparation.ipynb")
 │    └── 📂 VECTORS AND LANDMARKS             # Extended numerical data (Generate by "0 data preparation.ipynb")
 │
 └── 📂 MODELS                                 # Saved weights and models (.h5, .pkl)
      ├── 📂 INPUT - IMAGES                    # (Generate by "2 training from images data (CNN).ipynb")
      └── 📂 INPUT - VECTORS AND LANDMARKS     # (Generate by "3 training from vectors data (XGBoost).ipynb")
```
